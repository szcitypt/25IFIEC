## 这个项目是做什么的？

> 人的记忆是一种不讲道理的存储介质。这个项目存在的意义，就是把这些失衡的记忆碎片提取出来，完成从生物硬盘到数字硬盘的格式转换。

人的记忆分散、容易褪色，也很难按「文件夹」那样整齐归档。**MnemoTranscode**（简称 **MTC**，英文名也写作 *Memory To Code*）想做一件事：帮你把与重要的人、重要的时光有关的片段——文字、照片、声音、情绪、时间地点——**收进一套数字化的「关系档案」里**，并且用 **AI 协助整理、查找和重新讲述**，换句话说，是把脑海里难以按「文件夹」整齐归档的印象，变成在电脑里可保存、可检索、可分享的一种形式。

你可以把它理解成：**不是又一个聊天机器人，而是一个围绕「人与记忆」的档案 + AI 助手平台**——为谁建档、记下了什么、以后怎么找回、怎么用故事或对话重新体验那段关系，都由你和系统一起完成。

## 我具体做了哪些事？

在技术落地层面，我搭建并实现了一条可用的 **Web 全栈产品路径**，主要包括：

- **档案与成员**：为不同关系（如亲友、伙伴等）建立档案，管理其中的「成员」与结构化信息。  
- **记忆与多媒体**：记录记忆条目（含情感、时间、地点等元数据），配合后端与对象存储完成**媒体上传与私密存放**。  
- **语义检索**：接入向量数据库，支持用**自然语言搜索**相关记忆，而不是只会关键词匹配。  
- **AI 对话**：在**注入档案记忆上下文**的前提下与「角色化」对话交互（前端支持流式 / 打字机类体验）。  
- **故事书与时间线**：按记忆生成可读的生命叙事（多种风格），并提供可按时间、情感、成员等维度浏览的**时间线**。  
- **记忆胶囊**：定时解封类能力，用于「未来某一天再打开」的加密信件式体验。  
- **工程与基础设施**：前后端分离（例如 React + FastAPI）、数据库与缓存、容器化编排（Docker Compose）等，使上述能力可以**在本机或容器环境中跑通**。
