# 项目计划

## 目标

「瞬忆 · AI 记忆助手」用于技能节比赛展示，核心是呈现手机 APP、后端服务和 ESP32 硬件端之间的稳定联动闭环。

## 当前阶段

优先完成 mock 演示，不追求真实语音识别和真实音频上传。

当前交付重点：

- APP 页面完整、精美、可交互，并保留 Android APK 打包路径。
- 后端使用 Node.js + Express，返回固定的 mock 会议整理结果。
- ESP32 端使用 ESP-IDF，放在 `hardware/shunyi-esp32/` 下。
- 硬件端先上传测试文本，后端返回整理结果，APP 展示结果。

## 模块拆分

### APP

- 展示记忆卡片、会议整理、硬件连接状态和 mock AI 结果。
- 当前不申请真实麦克风权限。
- 当前不实现后台录音、静默监听、剪贴板监听、键盘捕获。

### Server

- 提供健康检查接口。
- 提供会议整理 mock 接口。
- 提供硬件上传测试文本接口。
- 预留真实大模型接口配置，但默认不启用。

### Hardware

- 使用 ESP-IDF 创建独立项目。
- 负责 ESP32 + INMP441 麦克风相关接线和采集触发演示。
- 当前阶段上传测试文本，不上传真实音频。

## 里程碑

1. 创建项目结构和文档。
2. 整理现有 APP 前端资源到 `app/`。
3. 初始化 Express mock 服务。
4. 初始化 ESP-IDF 项目骨架。
5. 打通 APP 调后端 mock 接口。
6. 打通 ESP32 上传测试文本到后端。
7. 准备比赛演示脚本和离线兜底方案。
